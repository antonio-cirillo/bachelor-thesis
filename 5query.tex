I grandi vantaggi economici derivanti dal paradigma del Cloud Computing ha portato molte aziende a partizionare i dati contenuti in un singolo database in più database, causando però un aumento della complessità per la gestione dei dati. La neccesità di manipolare dati provenienti da diversi database ha portato alla nascita delle \textbf{query distribuite}. Una query distribuita è un'interrogazione per l'acquisizione di dati memorizzati su diverse istanze di database. Esse permettono all'utente di effettuare operazioni di selezione, inserimento, modifica e eliminazione su più database contemporaneamente. L'utente che utilizza questo tipo di query è consapevole di interagire con più sorgenti dati ma il risultato che ottiene e la sintassi che utilizza sono analoghi a quelli di una classica interrogazione lanciata su un singolo database. Quest'idea è però un'astrazione di quello che è in realtà il comportamento delle query distribuite. L'esecuzione di una query distribuita su \textit{n} sorgenti  scatena l'esecuzione sequenziale di \textit{n} interrogazioni distinte su ogni singola sorgente. I risultati derivanti da queste \textit{n} interrogazioni vengono poi raggrupati e restituiti all'utente come singolo risultato. \\

\section{Obiettivi}
L'obiettivo all'interno di Fly è di permettere all'utente di utilizzare una normale query BSON per accedere a diverse sorgenti dati, siano esse locali o in Cloud. L'utilizzo delle query distribuite all'interno di Fly deve garantire un livello di astrazione tale da rendere il comportamento di una query distribuita omogeneo al comportamento di una semplice query su un singolo database, per questo motivo l'interazione coi diversi database e il raggruppamento del risultato verrà totalmente astratto all'utente.

\section{Implementazione}
L'implementazione delle query distribuite all'interno di Fly ha portato all'introduzione dell'entità \verb|distributed-query|. I parametri che compongono questa entità sono analoghi ai parametri dell'entità \verb|query| eccetto per l'aggiunta del parametro a numero variabile \verb|source_i| che identifica la \textit{i}-esima risorsa che verrà interrogata. La dichiarazione di una variabile di tipo \verb|distributed-query| è composta quindi dai seguenti parametri:
\begin{itemize}
    \item \textbf{query\textunderscore type}: stringa rappresentante il tipo di query da eseguire (\verb|select|, \verb|insert|, \verb|update|, \verb|replace| e \verb|delete|);
    \item \textbf{filter} (solo per operazioni di \verb|update| e \verb|replace|: interrogazione in linguaggio BSON per selezionare l'oggetto che si intende modificare, può essere specificata in due differenti formati:
    \begin{itemize}
        \item stringa di testo;
        \item nome di una variabile di tipo stringa.
    \end{itemize}
    \item \textbf{query}: interrogazione in linguaggio BSON da eseguire, può essere specificata in due differenti formati:
    \begin{itemize}
         \item stringa di testo;
         \item nome di una variabile di tipo stringa.
    \end{itemize}
    Come per l'entità \verb|query|, per le operazioni di inserimento, è possibile assegnare a questo parametro il nome di una variabile di tipo \verb|file|;
    \item \textbf{source\textunderscore i}: nome della \textit{i}-esima variabile di tipo \verb|nosql| rappresentante la connessione ad un database.
\end{itemize}
Il Listato~\ref{lst:dquery} mostra un'esempio di dichiarazione di un'entità \verb|distributed-query| che permette di selezionare i dati presenti in tre diversi database.
\begin{lstlisting}[language=FLY,caption={Dichiarazione di query distribuita di tipo selezione su tre database diversi.}, label={lst:dquery}]
var selezione = [type="distributed-query", query_type="select", query="Sintassi BSON", source_1=primoDatabase, source_2=secondoDatabase, source_3=terzoDatabase]
\end{lstlisting}

\section{Generazione codice}
Il programma Java, JavaScript e Python generato dal compilatore Fly per l'entità \verb|distributed-query| è anologo al programma generato per l'entità \verb|query|. Una volta dichiarata l'entità, in entrambi i casi, viene dichiarato all'interno del linguaggio l'oggetto relativo alla sintassi BSON da utilizzare durante l'esecuzione della query. Utilizzando il metodo \verb|execute()| sull'entità \verb|query|, come già introdotto nel capitolo precedente, il compilatore Fly utilizza uno dei metodi fra \verb|find()|, \verb|insertMany()|, \verb|updateMany()|, \verb|replaceOne()| o \verb|deleteMany()|, in base al tipo di operazione da eseuire, sulla variabile relativa alla collezione da interrogare. Questo funzionamento viene esteso alla chiamata del metodo \verb|execute()| sull'entità \verb|distributed-query| in modo da eseguire il metodo richiesto su tutte le collezioni specificate nei parametri \textbf{source\textunderscore n}. L'unica differenza implementativa sta nella gestione dei risultati. Nel capitolo precedente abbiamo visto i risultati ritornati dalle interrogazioni sui database MongoDB, in particolare la lista di tabelle per le interrogazioni di selezione e il numero di elementi rimossi per le interrogazioni di rimozione dei dati.

\subsection{Java}
All'interno del compilatore Java è stato introdotto il metodo \verb|__executeDis-| \verb|tributedQuery()|. Questo metodo consente di eseguire la stessa query su una lista di database MongoDB. Gli argomenti in input di questo metodo sono:
\begin{itemize}
    \item \verb|typeQuery|: argomento di tipo \verb|char| che indica il tipo di interrogazione da eseguire, i valori ammissibili sono:
    \begin{itemize}
        \item \verb|"S"|, per operazoni di \verb|select|;
        \item \verb|"I"|, per operazoni di \verb|insert|;
        \item \verb|"U"|, per operazoni di \verb|update|;
        \item \verb|"R"|, per operazoni di \verb|replace|;
        \item \verb|"D"|, per operazoni di \verb|delete|.
    \end{itemize}
    \item \verb|query|: oggetto Java rappresentante la query da eseguire;
    \item \verb|sourceList|: lista di tutte le risorse su cui eseguire l'interrogazione.
\end{itemize}
Questo metodo propaga l'esecuzione dell'interrogazione a tutte le risorse contenute nell'argomento \verb|sourceList|, inoltre, per le operazioni di \verb|select| e \verb|delete|, il risultato ottenuto dalle singole interrogazioni viene compattato e restituito all'utente come singolo risultato. Quindi, nel caso di un'operazione di selezione, tutte le tabelle ottenute dalle singole interrogazioni vengono unite e restituite in un'unica e compatta lista di tabelle, mentre, nel caso di un'operazione di rimozione, il conteggio dei vari elementi rimossi su ogni risorsa viene sommato e restituito all'utente. \\
Il codice Java generato dalla dichiarazione di un'entità \verb|distributed-query| comprende la dichiarazione degli oggetti MongoDB rappresentanti la sintassi BSON da utilizzare all'esecuzione dell'interrogazione e l'aggiunta di tutte le variabile rappresentati le risorse specificate all'interno di un'array. L'esecuzione del metodo \verb|execute()| comporta, all'interno del codice Java, la chiamata al metodo \verb|__executeDistributedQuery()| con il corretto valore assegnato all'argomento \verb|typeQuery|. \\
